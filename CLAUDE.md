# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Chrome Extension (Manifest V3) that analyzes LinkedIn feed posts and scores them 0-100% for likelihood of being AI-generated using heuristic pattern matching. No build step, no bundler, no dependencies — plain vanilla JS loaded directly by Chrome. All analysis runs locally (no API calls, no data leaves the browser). Published on the Chrome Web Store (v1.1.0).

## Architecture

**`detector.js`** — Core analysis engine and single source of truth. Exposes a global `AIDetector` object with:
- `analyze(text)` — returns `{ score, verdict, confidence, signals, wordCount }`
- `getScoreInfo(score)` — returns `{ min, bg, text, label, verdict, confidence }` tier for a given score
- `SCORE_TIERS` — array of score tier definitions (colors, labels, thresholds)

Contains six independent detection checks, each returning a partial score and signal list:
- `checkAIPhrases` — matches against curated phrase lists (high/medium tiers), capped at 30 pts. Medium phrase regexes are pre-compiled at init.
- `checkStructuralPatterns` — numbered lists, bullet points, emoji density, Hook→List→CTA formula, bro-poem style, capped at 35 pts
- `checkSentenceUniformity` — coefficient of variation on sentence lengths
- `checkHedgingAndFiller` — hedging phrase density per 100 words. Regexes pre-compiled at init.
- `checkEngagementBait` — regex patterns for CTAs like "Agree?", "Thoughts?"
- `checkRepetition` — repeated sentence starters (first 3 words)

Scores from all checks are summed and capped at 100.

**`content.js`** — Content script injected on `linkedin.com`. Uses `AIDetector.analyze()` and `AIDetector.getScoreInfo()` from `detector.js` (loaded first per manifest). Only scans on feed pages (path `/` or `/feed*`). Badges are `<button>` elements with ARIA attributes. Uses a single delegated click/keydown listener for all badge interactions. MutationObserver is scoped to the feed container when possible. No `setInterval`.

**`popup.html`** — Extension popup with a standalone text analyzer. Self-contained styles and inline `<script>`. Reads version from `chrome.runtime.getManifest()`. Supports Ctrl/Cmd+Enter to analyze.

**`styles.css`** — All badge and detail panel styles using BEM-like naming (`ai-detector-detail__signal-name`). Detail panel visibility is driven by `aria-hidden` attribute.

**`manifest.json`** — Manifest V3 config. Content scripts run at `document_idle` on `linkedin.com`. No special permissions required.

## Development

No build/lint/test commands — load the folder as an unpacked extension in `chrome://extensions/` with Developer Mode enabled. After code changes, click the refresh icon on the extension card and reload the LinkedIn tab.

Key DOM selectors for LinkedIn posts (may break if LinkedIn changes their markup):
- `.feed-shared-update-v2`, `[data-urn^="urn:li:activity"]`, `.occludable-update` (post containers)
- `.feed-shared-update-v2__description`, `.feed-shared-text`, `.update-components-text`, `.break-words` (post text)

## Key Conventions

- Score thresholds, colors, and labels are defined ONLY in `detector.js` `SCORE_TIERS`. Never hardcode them elsewhere.
- All content script styles go in `styles.css`, not inline. Only `background` and `color` on score-dependent elements are set via JS.
- Badge detail panel open/close is controlled via `aria-expanded` / `aria-hidden` attributes, which also drive CSS visibility.
- The `_metadata/` directory is auto-generated by Chrome and should not be modified.
- `screenshots.html`, `icons/generate.html`, and `linkedin-ai-detector.zip` are store-listing assets, not part of the extension runtime.
